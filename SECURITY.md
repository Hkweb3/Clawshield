# Security Policy

## Reporting a Vulnerability

Please report security issues privately if possible.
Email: Ks5134803@gmail.com

If the repository is hosted on GitHub, use the "Report a vulnerability" feature (Security Advisories).
If you cannot find a private channel, open a minimal public issue and request a private contact.

## Scope

We welcome reports for:
- Runtime guard bypasses
- Policy enforcement gaps
- Privilege escalation or unsafe file/network access
- Supply chain or dependency risk handling issues
- RCE or data exfiltration vectors

## Response Timeline

We aim to acknowledge reports within 3 business days and provide an update within 7 days.

## Supported Versions

Security fixes are applied to the latest release on the default branch.
